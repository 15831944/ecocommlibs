    • About MyScript Prediction SDK

    MyScript Prediction is designed for performing word completion and prediction. MyScript Prediction is an add-on MyScript Text feature. You can use it only if you have purchased the separate Prediction extension SDK.

    In brief, MyScript Prediction supplies:

    Item Description  Location
    API for C, Java, and .NET. Prediction API. edk/your_programming_language/
    ConjointRecognitionCompletionAndPrediction A full example of how to perform conjoint handwriting recognition, word completion and prediction on an ink sample. The predefined ink sample for this example displays the words “good aftern” (which could be completed as “good afternoon”). edk/programming_language/examples/prediction/

    MyScript Prediction has the following features:

    • Completion and prediction

      • Completion
        This works by an analysis of a word’s prefix. The handwriting recognition engine carries out recognition as an incremental process. It recognizes continuously, updating its result as new ink comes to hand. When you carry out handwriting recognition, you attach Linguistic Knowledge (LK) resources to the recognizer, from which it suggests the most likely candidates for the ink it has so far recognized. The recognizer produces a result for the sample that it has on hand. MyScript Prediction extension gets other elements in the attached resources (lexicons only) that have the same prefix. It finds words that have the same initial characters. These matches are highlighted in the candidate lists with a special flag so they can easily be retrieved and displayed.
        In the result, MyScript Prediction flags its completions with an omitted type flag. The “omitted” characters are those which have not yet been written! In the example here, the words “bank”, “band” and “banner” are suggested as potential “completions” of the initial word “ban”. The characters singled out in those words would appear with an omitted flag in the recognition result.

      • Prediction
        MyScript Prediction is also able to predict the next words, that follow the current word. It does this by using language models that can predict the most likely words that follow that word in common everyday language.
        This capability is known as “prediction”, rather than completion, as the program is suggesting something new, rather than completing some existing content.

    • Languages
      MyScript Prediction works with all languages supported by MyScript Text. You need to attach the linguistic resource (LK) for the required language. The same resource is used for recognition and prediction.

    • MyScript Text Input Method SDK
      MyScript Prediction also works with the MyScript Text Input Method SDK extension, where it is ideal for creating input methods and messaging-type applications.

    MyScript Prediction does not work with lexical unit type resources. It works with lexicons, though.

    Application blueprint

    The input is ink strokes or text strings. These are passed to MyScript Text recognizer. This latter carries out handwriting recognition and word completion/prediction on them. Omitted type flags will appear on certain result candidate objects after the recognition process. In other words, the “predicted” results are longer words which are seen as having some of their characters “omitted”.

    Depending on the properties you have set, you may retrieve completion and prediction candidate objects in addition to the usual recognition candidate objects. The prediction candidate objects are estimated based on the best recognition or completion candidate.

    Resources

    MyScript Prediction requires the following resources:

    Alphabet Knowledge
    MyScript Prediction requires an Alphabet Knowledge (AK) resource, as for Text recognition.

    Linguistic Knowledge
    You need to attach the linguistic resources (LK) for the required language. The same resource is used for recognition and prediction. You must attach the lk-text.res resource for the language in question, to the recognizer. You can also use any user-defined lexicon.

    You cannot use a lexical unit resource for a prediction task. Candidates from lexical unit resources cannot be flagged as omitted. However, they can be suggested as recognition candidates.

    Result

    The result consists of a list of candidates that are the proposed recognized samples, as described in MyScript Text SDK. In addition to the recognition candidates, the list includes potential completion candidates and also prediction candidates.

    Use cases

    The way you set up the Prediction process depends on what you want to carry out and the context of the application (which generally speaking will be an input method). The Prediction engine properties to use are listed below.

    Case 1: Completion plus prediction

    This use case is for input methods where the context of the recognition/prediction engine will not end with a space (e.g. a text string followed by strokes). The prediction engine can offer word completion on the existing word, and if required, prediction of the next word to come.

    Here is an example:

    Imagine an input method that displays suggestions.

    STEP 1: The context of the application contains several text words. In this example, the context is represented by the string “what ”.

    STEP 2: The user writes a new word. Potentially, the handwritten word may or may not be finished. In the example, the user enters the strokes of the word “time”.

    STEP 3: The context (“what ” entered via addString function) and the strokes (entered via addStroke function) are sent to the recognizer for processing. The result is then recovered and sent to the input method for display.

    STEP 4: The input method displays a suggestion list including recognition candidate objects (“time”), potential completion candidate objects (“timer”, “times”) and also prediction candidate objects (“time is”) . The prediction candidate objects are based on the best recognition/completion candidate (the candidate “time” in this case).

    To offer completion and/or prediction in this way, use these property settings:

    • set the wordCompletionCandidateListSize property to a value greater than 0. This property controls the number of candidate objects for completion that will be suggested by the recognizer.
    • set the wordPredictionCandidateListSize property to a value grater than 0. This property controls the number of candidate objects for prediction of next words that will be suggested by the prediction engine. If you do not wish to use prediction, set it to 0.

    Case 2: Prediction only

    This use case is for input methods where the context of the recognition/prediction engine will explicitly end with a space (e.g. a string ending with a space). In this case, completion is not required, as the existing word is already complete. You can however offer prediction of the next word to come.

    Here is an example:

    Again we have an input method that displays suggestions.

    STEP 1: The context of the application contains several words and is finished with a space. In the example, the context is represented by the string “what time”. The user does not enter any strokes.

    STEP 2: The context (“what time” entered via addString function) is sent to the recognizer for processing. The result is then recovered and sent to the input method for display.

    STEP 3: The input method displays a suggestion list including prediction candidate objects (“What time…is”, “What time…does”, “What time…will”). There is no completion in this case.

    To offer prediction in this way, use these property settings:

    • set the wordCompletionCandidateListSize property to 0. This property controls the number of candidate objects for completion that will be suggested by the recognizer. There will be no completion here so it must be disabled.

    • set the wordPredictionCandidateListSize property to a value greater than 0. This property controls the number of candidate objects for prediction of next words that will be suggested by the prediction engine. An average setting would be 3 or 4, so that several predictions can be displayed.

    We do not recommend using the addString function to send a string with a finishing space if it is not followed by addStroke: the engine will trim trailing space.
