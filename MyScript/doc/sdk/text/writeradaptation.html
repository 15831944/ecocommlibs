    • About MyScript Writer Adaptation

    MyScript Writer Adaptation extension automatically improves the handwriting recognition accuracy over time based on the analysis of the identified user’s writing style features. For this reason, it is intended to be used for single-user applications. It can be used for the development of interactive applications.

    MyScript Writer Adaptation is an add-on MyScript Text feature. You can use it only if you have purchased the separate Writer Adaptation extension SDK.

    In brief, MyScript Writer Adaptation supplies:

    Item Description  Location
    API for C, Java, and .NET. Writer Adaptation API. edk/your_programming_language/
    MyScriptWriterAdaptation.dll The specific library. engine/
    Resource files These resource files are specific to performing individualized handwriting recognition. resources/
    ConjointRecognitionAndAdaptation This code sample demonstrates how to create a profile object, a collector object and a trainer object, how to correctly attach the objects to each other and how to run the training and correctly store the results into the profile. edk/programming_language/examples/writeradaptation/

    MyScript Writer Adaptation has the following features:

    • Character level process
      MyScript Writer Adaptation works at the character level and does not rely on linguistic resources. The adaptation module leverages style characteristics of the characters written by the user to adjust the way recognition is performed.

    • Latin languages and cursive writing
      MyScript Writer Adaptation is limited to Latin languages and to cursive writing in the current version.

    • Improvement
      It improves the recognition accuracy gradually over time, after collecting samples from the recognized characters. In internal experiments, we have reached an average relative improvement of 40%.     Application blueprint

    MyScript Writer Adaptation SDK involves three different processes:

    • collection of individual handwritten samples
      This stage aims at collecting and storing recognized samples.

    • training upon collected samples
      During the training, the collected samples are exploited in order to output individual features.

    • adaptation of the recognizer
      In this last step, the individual features are used to improve the recognition.

    These three processes are not required to be run at every recognition pass. For instance, for the sake of time savings, API developers may choose to run training processes at dedicated time instants only.

    These three processes are handled through the use of three different objects:

    • Collector
      The Collector gathers recognized samples after a recognition process is done. The method commit allows sending the collected samples to the attached profile.

    • Trainer
      The Trainer processes the collected samples in order to improve the individual handwriting recognition by enhancing the Profile. The training can be launched through the method run.

    • Profile
      The Profile stores personal alphabet samples and other specific individual data (features, etc.) characterizing the writing style of a given user.

    You can call the three different processes enabled by MyScript Writer Adaptation in an asynchronous manner, as long as the dependency between the different objects are respected. The order of execution of the processes can vary depending on the characteristics of the device or the target application.

    Indeed, it may be interesting to run these processes only when required. For instance, the processing step of the Trainer requires a computational power and running time which may be inappropriate for devices with limited computational resources. In this case, only the collection of data may be activated. This data can then be processed by Trainer later on a different, more powerful devices (or alternatively on the same device, when in idle state).

    The three different scenarios are described in the following.

    Collection of individual handwritten samples

    The following figure depicts the guidelines to use, in the case that you need to run a collection only.

    To perform a collection of samples, MyScript Writer Adaption requires that:

    1. you create an engine.

    2. you create a recognizer, that is a StructuredInputRecognizer.

    3. you attach the resources to the recognizer, as for standard Text recognition (appropriate resources Alphabet Knowledge (AK) and Linguistic Knowledge (LK)). The data laid down in the resource files let the recognizerknow how to perform the segmentation of all the input ink parts. You can attach other resources which types are discussed later in this documentation.

    4. you create a new or load an existing Profile. The recognizer relies on personal data stored in the Profile, if existing, to improve the handwriting recognition. Indeed, the recognition process is adapted based on previously computed features, stored in the Profile. If there is no existing Profile, you need to create a new one.

    5. you create a Collector.

    6. you attach the Profile to the Collector.

    7. you attach the Collector to the recognizer.

    8. you attach the Profile to the recognizer.

    9. you send the ink to the recognizer through the setSource or the addStroke method.

    10. you run the recognition and the collection simultaneously via the run method. Once the recognition is completed by the recognizer, the Collector gathers the recognized character samples.

    11. you commit the collected samples from the Collector to the Profile thanks to the commit method.

    Training upon collected samples

    You need to perform trainings for building or improve already built Profile’s data. As a reminder, the training aims at exploiting the collected samples in order to output individual features.

    To perform a training, MyScript Writer Adaption requires that:

    1. you create an engine.

    2. you create a Trainer.

    3. you load an existing Profile. The Profile must exist already and it must contain collected samples, otherwise, the training serves no purpose.

    4. you attach the AK resource to the Trainer. The AK must be attached to the Trainer in order to enable the training process

    5. you attach the Profile to the Trainer. The AK AND the Profile must be attached to the Trainer.

    6. you run the training thanks to the run method. The last collected samples are then processed by the Trainer that outputs new personal handwriting features. Finally, the run method updates this data is into the Profile in view for the next recognition process.

    7. you save the Profile through the storeEngineObject method.

    Adaptation

    If the Profile already contains personal data, you can complete an adaptation without collecting data or processing samples. In this case, you only need to instantiate the recognizer and the Profile. Recognition will then benefit from the processed data contained in the Profile.

    To perform a recognition with adaptation, MyScript Writer Adaption requires that:

    1. you create an engine.

    2. you create a recognizer.

    3. you attach the resources to the recognizer, as for standard Text recognition (appropriate resources Alphabet Knowledge (AK) and Linguistic Knowledge (LK)).

    4. you load the Profile.

    5. you attach the Profile to the recognizer. The Profile must be attached to the recognizer.

    6. you send the strokes through the addStroke method, as for a standard Text recognition.

    7. you run the recognition with the run method. Given that you attached the appropriate resources (such as Alphabet Knowledge and Linguistic Knowledge) to the recognizer, recognition is performed using the extra information from the processed data contained in the Profile.

    Check the code sample provided (ConjointRecognitionAndAdaptation) with this SDK, in the folder edk/programming_language/examples/writeradaptation/ to see an example of use.

    Resources

    Alphabet Knowledge

    This resource contains the existing symbols for a given language. It is required as for standard Text recognition.

    Linguistic Knowledge

    This resource provides the information about the words, phrases and semantic unit existing in a given language. It is required as for standard Text recognition.

    For more details on the resources, check the dedicated section.

    Result

    The result of the collection and then, the result of the training performed through MyScript Writer Adaptation, are stored in the Profile. You must save this MyScript object (with the storeEngineObject function) in order to benefit from the adaptation in view for future recognition processes.
