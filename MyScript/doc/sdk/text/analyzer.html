    • About MyScript Analyzer SDK

    Analyzer SDK extension can identify different text layouts (paragraphs, bullet points, tables, and more) and text within geometric shapes. It is based on MyScript Analyzer technology. Analyzer is an add-on MyScript Text feature. You can use it only if you have purchased the separate Analyzer extension SDK.

    In brief, MyScript Analyzer supplies:

    Item Description Â Location
    API for C, Java, and .NET. Analyzer API. edk/your_programming_language/
    ank-standard.res The analyzer knowledge resource file. resources/analyzer/
    shk-analyzer.res A special shape recognition resource file, adapted to shape recognition in a note-taking context. resources/analyzer/
    DocumentAnalysis A full example of analysis of a document. The output is in the form of an SVG file. This is achieved using a parser that we provide. edk/programming_language/examples/analyzer/

    Application blueprint

    Once the MyScript environment is set through the engine creation, a document analysis with MyScript Analyzer SDK requires that :

    1. you create an analyzer object. Then you can attach a special analyzer resource file to this analyzer (ank-standard.res).

    2. you create a text recognizer, that is the StructuredInputRecognizer. Then you can attach the alphabet (AK) and linguistic resources (LK).

    3. you create a shapeRecognizer, a shapeBeautifier. Then you can attach the special shape resource file (shk-analyzer.res).

    4. you attach the structuredInputRecognizer, the shapeRecognizer, and the shapeBeautifier to the analyzer object.

    5. you create an analyzerDocument. Then you can feed it with the digital ink through the addStroke method.

    6. Finally, you run the analysis and get the result with the process method. During this analysis, the analyzer object separates the various elements that make up your document. It classifies them as categories of text objects (text-lines, paragraphs, lists) or as tables or shapes. These items are then passed to either the text recognizer or the shape recognizer, where they are converted. The result of the classification is stored in the analyzerDocument. The final result can be expressed in SVG format so it can be immediately used (check the provided code example).

    The analyzer works in a self-contained manner. However you still need to configure the recognizers (attach the Alphabet and Linguistic Knowledge resources that you wish to use, set any properties etc.) and attach them. The handwriting recognition engine, the shape recognition engine and their respective resources are therefore pre-requisites. If you do not attach them, the process will not crash: but you will get no labels for text or shapes. You will simply be able to retrieve the segmentation results of the analysis via the API.

    Only text and shapes can be recognized. Equations cannot be included at the current time (a correct result will not be obtained for them).
    There is no provision made for editing the results.

    Environment

    Resources

    MyScript Analyzer requires the following resources:

    • Analyzer Knowledge.This resource gathers the analyzer knowledge enabling the text/non-text classification.

    • Shape Knowledge. This resource is a special shape recognition resource file. This special resource is optimized for use with MyScript Analyzer.

    • Alphabet Knowledge. MyScript Analyzer requires an Alphabet Knowledge (AK) resource, as for Text recognition.

    • Linguistic Knowledge. MyScript Analyzer requires linguistic resources (LK) for the required language, as for Text recognition.

    Analysis mode

    The analysis performed by MyScript Analyzer can be split up into different steps:

    If you donât need the complete analysis, it is possible for the analyzer to carry out only parts of the processing.

    The API provides the parameter VO_ANALYZER_PROCESSING_LEVEL_VALUE that enables to specify the desired processing level.

    You can set it to the following values:

    Processing level Description
    STROKE_TYPE This is the simple text/non text classification. The process tags each stroke as text or non-text
    TEXT_LINE This is a layout extraction. The process adds the extraction of text-lines and the detection of text corrections.
    RECOGNITION This is the recognition stages for text strokes and shape strokes. The process also adds the extraction of paragraphs and lists. RECOGNITION implies the completion of the steps performed with STROKE_TYPE and with TEXT_LINE.
    BEAUTIFICATION This is the layout beautification. The process adds the beautification of shapes and text:
    - The height of the text is normalized.
    - The text-lines inside a paragraph or list are aligned on the left, and the inter-line is normalized.
    - The text inside a shape or a tableâs cell is aligned to the center of shape.
    - Paragraphs and lists are aligned with each other in the document.
    - The shapes are beautified.
    BEAUTIFICATION implies the completion of the steps performed with STROKE_TYPE, TEXT_LINE and RECOGNITION.
    The AnalyzerDocument object is a single page of digital ink. It will be updated by the analysis process to reflect the results. The analysis process is a non-incremental process. Each document is processed in its entirety.

    Analysis result

    After the analyzer has carried out the analysis, recognition and beautification of the analyzerDocument, the results are presented in different objects:

    The types shown in the diagram represent the items typically present in a handwritten document.

    Among others, the API provides the following types:

    Type Description
    TextLine This is a single line of text, with the properties:
    - label returned from the recognizer.
    - position and size represented by a bounding box, a baseline, and the height of the text.
    - orientation angle of the baseline.
    - justification property that represents either left or center alignment of the label in the bounding box.
    - underlines: the ranges of characters in the text-line that are underlined.
    - ink range
    Table This a group of cells with some text inside, with the properties:
    - number of rows and columns.
    - the position of the horizontal and vertical lines.
    - the position, size and orientation of each cell
    - ink range
    Shape This a single geometrical shape recognized by the MyScript Shape recognizer (for more details on the properties of a shape, please refer to the documentation delivered with the MyScript Shape SDK. )
    Group This a group of items brought together by a common characteristic:
    - PARAGRAPH: a coherent set of text-lines.
    - LIST: a bulleted list of text-lines.
    - TEXT_INSIDE_GRAPHIC: a text-line or a paragraph that is inside a shape or a tableâs cell.
    STOKE_TYPE This a tag assigned to each stroke of the document:
    - text: the stroke is part of a text-line
    - shape: the stroke is part of a geometrical shape
    - table: the stroke is part of a tableâs layout
    - underline: the stroke underlines some text
    - correction: the stroke is part of a text erasure (scratch-out, strike-through or cross-out)
    - erased text: the text stroke is erased by a correction
    - non-text: this tag is assigned after the text/non-text classification only. It identifies strokes that are not considered as text strokes.
