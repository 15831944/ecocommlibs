    • About MyScript Keyboard SDK

    MyScript Keyboard SDK provides you with the API for the keyboard service.

    In brief, MyScript Keyboard supplies:

    Item Description Location
    API for C Keyboard API edk/c/
    MyScriptKeyboard The specific libraries engine/bin/plateform-architecture/
    It is strongly recommended to read the documentation sections of MyScript Text since MyScript Keyboard makes use of its environment.

    Application blueprint

    MyScript Keyboard requires that:

    1. you create and configure a KeyboardLayout instance that describes your keyboard.

    2. you attach the KeyboardLayout to a StructuredInputRecognizer.

    3. you create a KeyboardInput and initialize it with the sequence of key commands to convey. You can send a fuzzyType compliant sequence or a smoothType compliant sequence.

    4. you set the KeyboardInput as the source of the StructuredInputRecognizer, run the recognition and get the result.

    The following describes the main objects of MyScript Keyboard environment in details.

    Environment

    The main objects of the API are described hereby.

    KeyboardInput

    The KeyboardInput is used to collect commands coming from your keyboard application.

    KeyboardLayout

    The KeyboardLayout gathers the description of your application keyboard layout. You can add as many keys as needed to your KeyboardLayout. Each key is represented by its physical location and a list of alternates, through the InputCharacter object. You can have different alternates per key such as in the case of a 12-key keyboard.

    InputCharacter

    An InputCharacter defines the alternates for a key of your keyboard. When adding an InputCharacter to the KeyboardLayout as a new key, you have to specify its top left coordinates (x,y), its width and its height.

    Configuring a keyboard

    Configuring a keyboard requires various steps:

    1. Create the KeyboardLayout object with the createKeyboardLayout method.

    2. Create an InputCharacter object to define a key. Define at least one alternate for this key using the addAlternate method. You can add other alternates. Set the alternate probability as you wish.

    3. Add the new key to the layout with the method addKey. Specify the location of the key (its top left (x,y)coordinates, its width and its height).

      Use the flag values VO_DISABLE_CASE_ALTERNATES and/or VO_DISABLE_ACCENTUATION_ALTERNATES to prevent the process from adding the associated case alternates and/or accentuation alternates to the layout.

    4. Add additional keys by repeating steps 2 and 3 until all keys have been added to the KeyboardLayout instance. You can reuse the same InputCharacter object for all the keys by using its clear method before adding alternate for the new key.

    Using fuzzyType

    FuzzyType allows the end-user to enter a string of characters by hitting one key after the other.

    MyScript Keyboard supports imprecise types. When the end-user misses the intended key and types a neighbor one, MyScript Keyboard uses linguistic information to recover from those mistypes and provides the expected word candidate anyway.

    Performing a recognition process on a fuzzy type input requires the following steps:

    1. Send each key stroke to the KeyboardInput via the pointerDown, pointerMove and pointerUp methods. Specify at each call of these methods the (x,y) coordinates and timestamp of the touch hits captured on the keyboard surface. The capitalization hint can be defined on the pointerDown event.

    2. Set the KeyboardInput as the source of the StructuredInputRecognizer with the setSource method.

    3. Run the recognition of the input with the run method.

    4. Retrieve the result using getResult method. Check MyScript Text section to know how to navigate through this result.

    Using smoothType

    SmoothType allows the end-user to type an entire word by drawing a smooth stroke over the keys composing this word.

    Recognition of a SmoothType input is only possible using MyScript’s text recognition and requires the following:

    1. Send the trajectory drawn over the keyboard layout by using the pointerDown, pointerMove and pointerUpmethods, providing the (x,y) coordinates and timestamp of each captured point. The capitalization hint can be defined on the pointerDown event.

    2. Set the KeyboardInput to the StructuredInputRecognizer with the setSource method.

    3. Run the recognition of the input using the method run, which invokes MyScript’s text recognition.

    4. Retrieve the result using getResult method. Check MyScript Text section to know how to navigate through this result.

    There is no need to press the space button between each entered word since one stroke stands for one word when using smoothType.

    Advanced use

    Capitalization

    MyScript Keyboard allows you to give a capitalization hint on pointerDownevents.

    Value Comments
    KEYBOARD_INPUT_CAPITALIZATION_AUTO MyScript Keyboard uses linguistic knowledge resources to choose the most appropriate case.
    KEYBOARD_INPUT_CAPITALIZATION_TITLECASE Forces the output to be in title case: if the letter is the first of a sentence it is upper cased. Otherwise, it is lower cased.
    KEYBOARD_INPUT_CAPITALIZATION_UPPERCASE Forces the output to be in upper case.

    Boosting alternates

    You can set the alternateProbability parameter of a key alternate with the method addAlternates to boost its probability of selection over the other key’s alternates.

    By default, when you add one alternate, MyScript Keyboard also implicitly generates the related accentuated lower case alternates, the related upper case alternates and the related accentuated upper case alternates for the key. Generated alternates depend on the Alphabet Knowledge resource attached to the StructuredInputRecognizer. Say you target French text recognition, you add alternate “a” to one key. Then alternates “à”, “â”, “ä”, “À”, “Â” and “Ä” are also generated for the key.

    You can rely on MyScript Keyboard models to automatically set the probability of selection of the alternates by setting the alternateProbability parameter to 0.0f.

    Or, you can choose which alternate to boost over others by setting a probability value from 0.0f to 1.0f (the closest to 1.0f the higher the probability).

    Setting alternateProbability to 0.0f is the recommended option since it assigns the variant probabilibites that maximizes the accuracy in most of the use cases.

    Number and accents

    Use the addCharacter method of the KeyboardInput object whenever:

    • your application uses an alternate layout for digits and symbols.

    • your application displays overlay accent panels (as offered after a long press, in common applications).

    • out-of-dictionary words are entered, since MyScript Keyboard does not support fuzzy or smooth typing on out-of-dictionary entries.

    Method Description
    AddCharacter Adds a digital text character.

    CJK languages

    This section gives details on keyboard settings for Chinese, Japanese and Korean languages.

    Check also Linguistic Adaptation with transliteration and Transliteration since there are some specificities for those cases.

    Chinese

    Using Stroke Input Method (Wubihua)

    The Stroke Input Method (also called Wubihua method or Bihua IME) refers to a Chinese input method for keyboards. Five keys are each assigned a certain type of stroke. Any word is defined as an ordered series of stroke types that can be entered using these five keys. The figure below shows the five categories of strokes, the associated code points and stroke examples.

    To trigger the Stroke Input Method, do the following:

    1. Configure the KeyboardLayout as described in Configuring a keyboard. Add the five keys using addKeymethod.

      Make sure you provide the exact code points specified by MyScript for these five keys. Check the code points in the illustration above.
      You can also add the any key to the keyboard, that stands for any of the five keys.

    2. You have set the KeyboardInput as the source of the StructuredInputRecognizer.

      Make sure you attached an Alphabet Knowledge resource that supports the five stroke’s code points.

    3. Send a fuzzyType compliant sequence. Use the method addTransliterationBreak to notify the end of a stroke sequence standing for one symbol.

      You can omit this break marker between two symbols inside a word when this symbol combination can only stand for one unique possible word. In such a case, the engine will automatically determine the end of symbols.

    4. Run the recognition as usual and get the result.

    Using pinyin

    Pinyin input is triggered once you have configured your keyboard with the entire set of QWERTY keys and you have attached the appropriate resources, that are Simplified Chinese (zh_CN) Alphabet Knowledge and Linguistic Knowledge resources.

    You can use abbreviated syllables as input: use the first symbol of every syllable in the word. The engine automatically recognizes the intended word.

    Japanese

    Hiragana and Katakana keys can be added to the keyboard following the same procedure as described in Configuring a keyboard.

    Korean

    Hangul Jamo keys can be added to the keyboard following the same procedure as described in Configuring a keyboard. Make sure you also have attached the appropriate resources, that are Korean (ko_KR) Alphabet Knowledge and Linguistic Knowledge resources.

    We strongly recommend using the Hangul Compatibility Jamo unicode, though simple Hangul Jamo is also supported.

    You can decide not to include the exhaustive Jamo letters to your keyboard. In particular, your keyboard may omit some compound vowels. In such case, the engine supports the substitution form (by typing the sequence of simple Jamo vowels standing for the compound vowel). This is supported for the following four symbols only:

    • ㅑ: [ㅏ, ㅏ]
    • ㅕ: [ㅓ, ㅓ]
    • ㅛ: [ㅗ, ㅗ]
    • ㅠ: [ㅜ, ㅜ ]
    Word recognition using substitution forms of compound vowels is ensured only when the intended compound vowel does not have its own keyboard key, as illustrated below.

    Piping input text

    You can pre-fill a KeyboardInput using theaddString method. This is useful when a field with existing content gains focus or when you want to move the cursor between two already recognized words.

    Method Description
    AddString Adds a digital text string.

    Spelling correction

    You can overcome spelling errors by using the spelling correction property.

    The recognizer will guess a character that is missing, or ignore a character that is present, to find a lexical match for the input. A flag will appear in the result to warn you when these events have occurred. Use spelling correction if you want to know when the writers have added or omitted characters in relation to the lexical entries involved.

    This is a value between 0 and 255: the strength of the property depends on the value you define. 0 means that no correction is allowed, so no characters will be substituted or flagged as omitted: 255 means that maximum distortion is allowed, so substitutions are freely allowed. In practice however, there are 4 “key values” that you should use, 0, 50, 100 and 150 (other values will be rounded to the nearest key values). The flags associated with the spelling corrections are:

    Flag Description
    SUPERFLUOUS The candidate contains at least one superfluous character.
    SUBSTITUTED The candidate contains at least one substituted character.
    OMITTED The candidate contains at least one omitted character.

    Here are some examples:

    • OMITTED: Input is “trafagar”. The word level candidate is “trafalgar” but the “l” candidate has been flagged. The only way for the input “trafagar” to be read as “trafalgar” is to imagine that one letters has been omitted. “l” is inserted into the result and you are warned with an “omitted flag” on the candidate.

    • SUBSTITUTED: Input is “trafalwar”. The word level candidate is “Trafalgar”, but the “g” candidate has been flagged. Here, the presence of letter “w” would not ensure a lexical match, so the candidate “g” is chosen, and substituted at that place. It swaps “w” for “g” in the result but it does warn you with a “substituted flag”.

    • SUPERFLUOUS: Input is “trafalggar”. The word level candidate is “Trafalgar”, but you can see that the input contains more characters than the lexicon entry. Consequently, the second “g” candidate has been flagged as “superfluous” and that character does not appear in the final word candidate.

    Candidate flags

    You can check the candidate flags to know whether the candidate was generated following fuzzy typing or smooth typing. Here are the three main flag values:

    Candidate flag Description
    CANDIDATE_FLAG_KEYBOARD_RAW The candidate corresponds to the raw input.
    CANDIDATE_FLAG_KEYBOARD_FUZZY The candidate corresponds to the input interpreted as fuzzy typing.
    CANDIDATE_FLAG_KEYBOARD_SMOOTH The candidate corresponds to the input interpreted as smooth typing.
