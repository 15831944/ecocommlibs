MyScript Gesture

  • SDK
  • Gesture

    • About MyScript Gesture SDK

    MyScript Gesture SDK provides you with the API for gesture recognition.

    In brief, MyScript Gesture supplies:

    Item Description Location
    API for C Gesture API edk/your_programming_language
    MyScriptGesture The specific libraries engine/
    It is strongly recommended to read the documentation sections of MyScript Ink and MyScript Document since MyScript Gesture makes use of their environments.
    Only scratch-out gesture can be recognized when using MyScript Gesture SDK with MyScript Ink1.
    MyScript Gesture can be used when recognizing content compatible with MyScript Text and MyScript Shape.

    Application blueprint

    MyScript Gesture requires that:

    1. you create and set an InkSampler instance that gathers digital ink.

    2. you attach a GestureRecognizer to the InkSampler and set it.

    3. you attach and set a GestureHandler to the GestureRecognizer and set it. You can provide the InkSampler with the input ink and start the recognition process.

    4. you get the result when the GestureHandler notifies you that the recognition result is available.

    The following describes the main objects of MyScript Gesture environment in details.

    Environment

    The main objects of the API are described hereby.

    InkSampler

    The InkSampler gathers:

    • the target. This refers to the ink context, that is to say, the existing digital ink which potential gestures will apply a change on.

    • the input ink. That is the potential gesture that will induce the target to change.

    GestureRecognizer

    The GestureRecognizer processes the input ink. It decides whether this ink is just an additional ink stroke or a gesture. In that case, it also identifies the type of gesture corresponding to the input ink. Its decision is stored in a GestureResults object.

    GestureHandler

    The GestureHandler provides callback function types to notify your application when the GestureRecognizer has released its result or to notify you when it is added to or removed from a GestureRecognizer.

    GestureResults

    The GestureResults object stores the list of candidate gestures fitting the input ink. They are ranked from the most likely to the least. Each candidate is a GestureResult object that you can access.

    GestureResult

    The GestureResult object gathers the details regarding a candidate gesture. It specifies the nature of the candidate in details (gesture intent, type), confidence scores, length of gesture symbol, number of strokes of gesture symbol, portion of ink concerned by a potential change, etc.

    Starting a session

    Before starting a gesture recognition session, you need to load the digital ink gathered from the acquiring device. This ink is the target, that is to say the ink context the gesture will apply potential changes on. We strongly recommend to set the ink to a page format 1.

    The target ink, whatever its type is, MUST contain TAGS. This is automatically the case if this ink has been processed by a Text or a Shape recognizer before starting the Gesture session. Otherwise, you need to make sure the structure contains tags such as “word”, “char” or “text” for each corresponding ink portion.

    Then, starting a gesture recognition session requires various steps:

    1. Create the InkSampler object with the createInkSampler method. Provide it with the target ink1 or document’s page.

    2. Register the device and the pointer that will be used to enter the input ink through the methods registerDevice and registerPointer.

    3. Create the GestureRecognizer object with the method createGestureRecognizer.

    4. Attach the GestureRecognizer to the InkSampler through the method setHandler.

    5. Create the GestureHandler object to enable your application to receive updates regarding the gesture recognition task. To do so, you need to create a UserObject and to register it as a VO_IGestureHandler through the method registerInterface.

    6. Attach the GestureHandler to the GestureRecognizer through the method setHandler.

    Processing a flow of points

    Once you have started the session, you can provide the input ink in view for gesture recognition.

    Each point of the input ink can be sent to the InkSampler object:

    1. Send the first point of a stroke. The first point of a digital stroke is always a pen down point. Use the method pointerDown to send the point to the InkSampler object.

    2. Send the next points of this stroke, if any. For each new point to add, use the method pointerMove to send them to the InkSampler object.

    3. Send the last point of this stroke. The last point of a digital stroke is always a pen up point. Use the method pointerUp to send the point to the InkSampler object.

    4. You have called the method pointerUp. From this instant, a timer is triggered. By default, it is set to 500ms. You can change this value according to your needs, through the variable VO_GESTURE_DECISION_DELAY, in the GestureRecognizer property set.

    5. If you send a new sequence of points (a new stroke), before the timer delay, starting by using the pointerDown method, the timer is stopped. It is reset when you call the next pointerUp method.

    6. If there are no other calls to the pointerDown method before the delay, the timer goes on. After the delay, the GestureRecognizer delivers its decision. The GestureHandler then notifies your application that a result is available through the callback method onResults. You can implement this callback method according to your needs.

    Getting the result

    You have started the session and sent the input points. The onResults function has notified your application that the recognition result is available. You can now get the result and parse it according to your needs.

    The recognition result is a GestureResults object which is a list containing different GestureResult objects. You can access different pieces of information from the recognition result, following various steps:

    1. Get the number of candidate gestures in the GestureResults using the getCount method. Access a single candidate using the getAt method.

    2. From the GestureResult interface, you can access different fields that describe this candidate. Get the intent of this gesture using the getIntent method. The intent is the general category of gesture that is meant to be executed. For instance, it can be an insertion, or a selection, or an erasure.

    3. Get the type of this gesture using the getType method. The type details the nature of the gesture. Say the intent is an erasure, the type can be a cross-out.

    4. Get the ink selections to be modified by this gesture. You get the selected ink portion(s) concerned by a modification, in the format of a InkSelection object, if you have configured your InkSampler with a ink 1 object at the starting stage. It is in the format of a pageSelection, if you have configured your InkSampler with a page1 object. Use the getSelection method to obtain the selection. Depending on the type of the recognized gesture you get the ink selection before and/or after and/or on the gesture using the methods getBeforeSelection, getAfterSelection and getSelection respectively.

    The ink selection locations are switched for right-to-left written languages such as Arabic languages. before selection is on the right of the gesture and after selection is on the left, in that case.

    The following table gives the description of the supported intents:

    Intent label Description
    ADD refers to an addition of actual ink strokes
    ERASE refers to a gesture aiming at deleting a portion of ink
    SELECT refers to a gesture aiming at choosing a portion of ink
    INSERT refers to a gesture aiming at putting something in-between two ink portions
    JOIN refers to a gesture aiming at connecting two ink portions
    DECORATE refers to a gesture aiming at emphazing an ink portion (e.g. through a line beneath it)

    Advanced use

    Getting the scores

    The GestureRecognizer object outputs a list of gestures fitting the input ink. It ranks these gestures according to a score. It indicates the possible confusion that may exist between the different other gestures of the list. It ranges lies within the range of [0-1], 1 being the highest level of confidence.

    From the interface of the GestureResult, you can obtain this score using the method getScore.

    Rejecting a recognition result

    Once you have obtained the recognition result, using onResult method, you can accept or reject the gesture result. To do so, implement the callback method onResult so that it returns a negative value when you want to reject the gesture.

    When onResult returns a negative value, MyScript Gesture sends you back a new GestureResults from the all the input strokes but the very last one. If you accept this new result, then the last stroke is kept and returned in a second GestureResults later on. Otherwise, the strokes are lost.

    Getting the gesture ink range

    The InkSampler object gathers the target and the input ink. It considers the input ink as a set of strokes among which are the strokes forming the recognized gesture. You can access the strokes forming the recognized gesture through the API methods, from the GestureResult interface:

    Method Description
    getFirstStroke returns the stroke index that the gesture starts from
    getLastStroke returns the index of the last stroke associated to the target gesture

    Getting notified when ink is added/removed

    You can get notified when strokes are added or removed from the InkSampler. The InkSamplerListener provides callback function types to notify your application in such cases.

    Here are the main callback methods provided in the interface of the InkSamplerListener:

    Method Notification case Returned value
    onSampleAdded a sample point (pen up or pen down) has been added into the InkSampler returns the sample point index and stroke index in which it is added
    onStrokeDone a pen up point has been added into the InkSampler, indicating that it is the last point of the stroke returns the index of this stroke
    onStrokeDetached a stroke has been detached (erased) from the InkSampler returns the index of the detached stroke

    To use the InkSamplerListener, attach it to the InkSampler through the addListener method from the IInkSampler interface.

    You can attach several InkSamplerListener to the same InkSampler.

    Getting geometry labels

    The GestureResult object provides a geometry field that describes the way the gesture is drawn. It can be down then left for the return gesture, for example. You can obtain this data through the method getGeometry from the GestureResult interface.

    The following table shows the existing geometry labels with the associated gesture intents and types, with the possible actions in Ink environment.

    Intent name Type labels Geometry labels Possible actions in Ink environment
    ADD NONE UNDEFINED Add strokes
    ERASE SCRATCH_OUT SCRATCH Erase strokes

    The following table shows the existing geometry labels with the associated gesture intents and types, with the possible actions in Document environment.

    Intent name Type labels Geometry labels Possible actions in Document environment
    ADD NONE UNDEFINED Add strokes
    ERASE SCRATCH_OUT SCRATCH Erase words and characters when gesture is applied on text content. Erase shapes and primitives when gesture is applied on shape content. Erase strokes the gesture is applied on, in any other case.
      STRIKE_THROUGH SLASH, BACKSLASH, LEFT, RIGHT Erase words and characters when gesture is applied on text content.
      DOUBLE_STRIKE_THROUGH LEFT-LEFT, RIGHT-RIGHT Erase words and characters when gesture is applied on text content.
      CROSS_OUT CROSS Erase words and characters when gesture is applied on text content.
      PIGTAIL_OUT PIGTAIL Erase one character when applied on text content.
    SELECT ELLIPSE ELLIPSE Select full words when gesture is applied on text content. Select full shape and primitive when gesture is applied on a shape content.
      FREE_FORM FREE_FORM Select full words when gesture is applied on text content. Select full shape and primitive when gesture is applied on a shape content.
    DECORATE UNDERLINE LEFT, RIGHT Underline words and characters when gesture is applied on text content.
      DOUBLE_UNDERLINE LEFT-LEFT, RIGHT-RIGHT Underline words and characters when gesture is applied on text content.
    JOIN ARC_JOIN BOTTOM_ARC Join line portions when gesture is applied on text content.
      I_BEAM UP Join line portions when gesture is applied on a space in text content.
      SCRATCH_OUT SCRATCH Join line portions when gesture is applied on a space in text content.
    INSERT I_BEAM DOWN Insert a space between characters when gesture is applied on text content.
      RETURN DOWN_THEN_LEFT, DOWN_THEN_RIGHT Insert a line feed between characters/words when gesture is applied on text content.
      V_INSERT V_TO_LEFT, V_TO_RIGHT Insert a space between characters when gesture is applied on text content.

    1. MyScript Gesture is compatible with MyScript Ink. However, you will fully benefit from MyScript Gesture features if you define the target ink as a page object, which belongs to Document Model API. Indeed, MyScript Gesture performs better on page objects than on ink objects because it can access and take advantage of data computed during recognition tasks.
