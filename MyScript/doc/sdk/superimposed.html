MyScript Superimposed

  • SDK
  • Superimposed

    • About MyScript Superimposed SDK

    MyScript Superimposed SDK uses MyScript Superimposed technology to enable the recognition of letters, words or parts of words written one over the other, without any explicit separation between consecutive fragments (it is also called superimposed handwriting).

    In brief, MyScript Superimposed supplies:

    Item Description  Location
    API for C, Java, and .NET. Superimposed API edk/your_programming_language/
    Resource files These files contain specific data needed to perform superimposed writing recognition. resources/
    Code sample This code sample shows how to perform a recognition of superimposed text edk/your_programming_language/example/text/

    MyScript Superimposed workflow is exactly the same as MyScript Text’s. We highly recommend to read MyScript Text documentation beforehand. In particular, the following sections are fundamentals:

    Application blueprint

    Once you have set the MyScript environment through the creation of the engine object, a superimposed text recognition with MyScript Superimposed SDK requires that:

    1. you create a recognizer. The engine requires a recognizer specialized for text recognition. The recognizer will process the input digital ink as a series of strokes. You can use a voStructuredInputRecognizer.

    2. you attach specific resources (AK and LK in the diagram above, and optionally SK) to the recognizer. The data laid down in the resource files let the recognizer know how to perform the segmentation of all the input ink parts.

    3. you send the strokes to the recognizer that your application has gathered from user handwriting with an electronic pen or a smartphone, tablet or an interactive whiteboard. The recognizer expects characters, portion of words, or words to be written one over the other.

    4. you start the recognition up. The recognizer is based on three conjoint processes: character classification, segmentation and linguistic-based analysis. These three main stages take place simultaneously. Information is exchanged between the three processes to find the most probable recognition result.

    5. you get back the recognition result in the form of a list of characters that could match the input ink.

    Environment

    Input handwriting styles

    The end-user can write one word fragment over the other or one character over the other, as depicted below. Spaces between two written words are automatically added, so that there is no need for an explicit gesture.

    Input digital ink from the end-user are sent to the recognizer via the addStroke method. However, it is possible to import digital characters thanks to the addCharacter or addString methods. By doing this, you can retrieve an existing result and import it.

    MyScript Superimposed enables input of word fragments over the others. However, make sure delayed strokes are entered together with the word fragment they belong to. For instance, the word “attach” is written as two fragments, in the figure below: first word fragment is att and second word fragment is ach. The delayed stroke, in red, must come right after the first fragment (not after the end of the word!).

    Alphabet Knowledge

    It is compulsory to attach an Alphabet Knowledge to the recognizer. This resource file gathers all the existing symbols for a target script.

    There is one AK resource per supported language. AK files follow the naming convention: [language+country_prefix]-ak-superimposed.res. For instance, Spanish AK resource for superimposed is es_ES-ak-superimposed.res.

    Linguistic Knowledge

    Also denoted as “LK”, this optional resource file supplies information about the words, sentences or particular semantic units (such as dates, phone numbers and so on) that may exist in the target language. This resource differs for each language. For instance, for United Kingdom English, the LK resource is en_GB_lk-text.res. The LK resources required by MyScript Superimposed are exactly the same as the MyScript Lingo package LK resources since they cover both cursive and superimposed writing styles.

    For more details on the LK resources delivered with each language pack, see the dedicated page in MyScript Text documentation.

    Subset Knowledge

    This type of resource file is optional. It is a subset of the symbols existing in the Alphabet Knowledge. The Subset Knowledge (SK) reduces the set of symbols eligible as recognition results.

    MyScript Superimposed comes with SK source files, that you can compile, for each of the supported languages.

    When dealing with one language only, it is highly recommended to attach the SK corresponding to the target language since this increases the recognition accuracy and this avoids confusion with other characters.

    An SK could also be helpful when dealing with a field that accepts only numbers (postal codes, an ID numbers, etc.). You can build custom SK resources through tools provided by MyScript.

    Check these SK files to know the set of characters supported by MyScript Superimposed.

    To learn more about the resource types, see MyScript Text SDK section.

    Recognizer

    The recognizer processes the input digital ink and outputs the recognition result. You can tune the recognizer performances. For instance, the recognizer speed can be adjusted, as described here. Refer to the section Optimizing speed and accuracy in MyScript Text for other tuning features.

    Recognition result

    The recognition result consists of a list of candidates that are the proposed recognized symbols. Candidates are ranked in the list according to scoring metrics. Finally, input ranges link the recognition result elements to their counterparts in the input ink.

    Running a recognition

    You can rely on the steps of MyScript Text recognition process to run a recognition.

    Parsing the result

    After the recognition process, the recognizer outputs the result. The structure of the result is of the same kind as the result delivered after text recognition with MyScript Text. The recognizer outputs a list of propositions that match the input ink. These propositions are called candidate objects. Candidate objects are retrieved by an iterative process at several different levels: character, word and text.

    Each candidate is assigned with scores that express how confident the recognizer is with this proposition. You can read more about the result here.

    Steps

    After the recognition process, you can explore the resulting list of candidates and go across its elements following the steps hereunder:

    1- Get the result
    You obtain the recognition result after the recognition process thanks to the getResult method from the voIRecognitionResultProvider interface.

    2- Access the candidate objects
    From the RecognitionResult interface, you can use getCandidates to get an iterator over the different candidate objects.

    3.- Access recognition data per candidate
    Once you have a candidate, you can access its associated recognition data:

    • the label with the getLabel method,

    • the resemblance score with the getResemblanceScore method,

    • the normalized recognition score with the getNormalizedRecognitionScore method,

    • the segments with the getSegments method. A part of the input that makes up the candidate’s segmentation is referred to as a segment.

    • the flags with the getFlags method, that notifies a particular issue with the candidate (omitted, erased, unknown, substituted, superfluous or emphasized character, word out of lexicon, etc.)

    • etc.

    Getting the input range

    The input range value refers to the notion described in MyScript Text. The input range defines parts of the input in relation to the result. You can reach the input range of characters thanks to the getInputRange method, from the interface of the SegmentIterator.

    Advanced use

    Recognizing gestures

    By default, MyScript Superimposed SDK supports the recognition of gestures (left to right and right to left). This capability is ensured through the data embedded within the provided Alphabet Knowledge (AK) resource (that is specific to Superimposed SDK) and within the provided Linguistic Knowledge (LK) resource.

    Carriage return management

    By default, the provided resources do not allow the recognition of the carriage return gesture (down then left). To support the carriage return gesture, you have two options:

    • you can build your custom LK resource. Then you will add the carriage return code point in this new custom resource. Finally, you will attach this custom resource to the recognizer, in addition to the provided AK and LK resources.

    • or, you can edit the mul-lk-gesture.res. Then you can add the code points of the gestures to recognize1 and attach this resource to the recognizer, in addition to the provided AK and LK resources.

    Learn how to build a custom resource in the section Creating custom resources.

    Space

    By default, the superimposed engine will automatically insert spaces where they make sense in the input sequence. You may want to add a dedicated UI key to let users force space insertion or potentially rely on a left-to-right gesture.

    To enable the left-to-right gesture, you will need to attach mul-lk-gestures.res and add the left-to-right gesture. Note however that this gesture may introduce confusion, for instance with the input of dashes.

    In the case of right-to-left languages, such as Arabic, you may want to rely on the right-to-left dash gesture to represent backspace.

    Backspace

    Backspace symbol (right-to-left dash gesture) can be recognized when they are written alone. In other words, when the user writes one or several right-to-left strokes alone, they will be interpreted as backspaces. A right-to-left stroke is never interpreted as a backspace as soon as it is in the middle of a word or a sentence, unless the word or the sentence has been transmitted to the engine via the addString or the addCharacter methods. However, if you attach the mul-lk-gestures.res and add the right-to-left gesture, it is possible to recognize backspace symbols in the middle of words1.

    In the case of right-to-left languages, such as Arabic, you may want to rely on the left-to-right dash gesture to represent backspace.

    Extensions

    MyScript Superimposed SDK relies on MyScript Text technology to enable the recognition of superimposed handwriting.

    Some of MyScript SDKs can be used as extensions to MyScript Superimposed SDK:

    • MyScript Text Input Method
      MyScript Text Input Method enables to carry out background handwriting recognition processing in a separate thread. It can be used for the development of interactive applications.

    • MyScript Prediction
      MyScript Prediction is an extension SDK to the MyScript Text handwriting recognition system. It is designed for providing word completion and next word prediction capabilities. MyScript Prediction is compatible with MyScript Superimposed. You can thus attach a “predictor” object to the recognizer. This last carries out both handwriting recognition and word completion/prediction. The recognition result appears as usual, but due to the presence of the Prediction engine, omitted type flags will appear on certain result candidates. In other words, the “predicted” results are longer words which are seen as having some of their characters “omitted”. Depending on the properties you have set, you may retrieve completion and prediction candidates in addition to the usual recognition candidates. The prediction candidates are estimated based on the best recognition or completion candidate.
