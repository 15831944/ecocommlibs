Overview

Learn the basics of MyScript Technology and get the basic principles of handwriting recognition as handled in MyScript.

  • Technology
  • Overview

    • Why support handwriting?

    Writing is defined as a communication way that is based on an organized system. A writing system relies on a set of symbols whose combinations give rise to different possible meanings. The ordering of these symbols or combinations of symbols is usually ruled by a shared grammar and syntax knowledge that enable the construction of meaningful messages. Handwriting refers to writing meaningful symbols or combination of symbols by hand, based on a shared set of grammatical rules.

    Among the many writing-based communication modes, handwriting remains the most natural, familiar and flexible way to enter information. We are accustomed to writing on sheets of paper which enables us to efficiently and naturally handle a wide range of contents from simple text to diagrams, music scores, mathematical formulas, etc. An ideal input method should embed all these features for ensuring a quality user experience.

    The main limitation of handwriting concerns the restricted recording and processing possibilities as compared to digital data. The only way to couple the suppleness and the naturalness of handwriting with the power of digital technology is to bring forth Handwriting Recognition (HWR).

    The key idea in handwriting recognition consists in taking handwritten information and converting it directly into computer readable information with minimum human intervention.

    Input

    We usually need a medium to write down a message by hand and we usually use traditional tangible ink to do so.

    Suppose you want to keep going with traditional ink, then you need a way to first convert this ink to a computer-processable format in order to go on with the handwriting recognition. The usual way to do that is scanning a color, black and white or gray-scale written page, by using an optical scanner or a camera. This is called offline data and the recognition method used in this case is usually referred to as OCR (Optical Character Recognition) for printed text, and it is called ICR (Intelligent Character Recognition) for handwritten data.

    OCR works purely through a visual representation of text and does not refer to any dynamic information, such as how the character was written and in which order. It also has difficulty in separating the handwritten information from whatever else is printed on the paper (lines, boxes, combs, machine printed text, etc), which leads to confusion and errors.

    Capturing the trajectory of handwritten strokes can provide you with this dynamic information. This is possible thanks to the digital ink. As opposed to OCR, in MyScript technology, we will refer to this approach as on-line recognition.

    Digital ink

    The stroke of the digital ink is defined as the trajectory of the pen from the moment it is touching the surface (pen-down) until it is lifted again (pen-up). A stroke is a sequence of 2D points (x,y) ordered by time. Digital ink can be defined as a series of strokes, but note that this is a dynamic process that takes into account where strokes start and end and in which order they were written.

    Some hand printed characters are made of one stroke (for example, “o”), others are a combination of several strokes (such as “k”), and in cursive handwriting, a word can be written in a single stroke.

    Getting digital ink

    You can capture digital ink through a digital pen or pointing device. However MyScript is just a recognition software and does not provide with automatic features for extracting digital ink from your device. There is a variety of pen and pointing devices on the market, such as the Anoto Digital Pen, smartphones, tablets, interactive whiteboards, etc.

    To use your digital ink with MyScript, we suggest the following strategies:

    • use the InkTool utility to directly run recognition with the digital pen . InkTool has an in-built recognition process that relies on your SDK installation (this is also an excellent way of testing that you have correctly installed the MyScript SDK). If you are new to MyScript, this is a good way to learn how MyScript works and how to set up resources and constraints to get the best results, without writing a single line of code and directly using the digital ink files from your capture device.

    • Develop an application or modify our examples to run recognition. This second strategy is useful to learn how our API works and how you can integrate our technology in your applications.

    Ink file format

    The ink file (.ink) format stores the digital ink strokes. This is an ASCII text file containing a sequence of values separated by spaces. The file contains the number of strokes, and for each stroke its number of points, then for each point its x and y coordinates.

    Here is an example of such a file:

10        // Number of points in the first stroke

    Resources

    We previously defined handwriting. To sum up, what we want to convert into digital text is:

    • a set of symbols or combinations of symbols,
    • that are ruled by norms, meaningful,
    • and written by hand, through digital ink

    Based on this, MyScript will need the following items to perform handwriting recognition:

    • a set of resources
      The resources will model all existing symbols of a writing system on one hand, all the ordering rules that confer meaning to a message on the other hand. MyScript relies on binary resource files (with .res extension).

    • an ink processor i.e. a recognizer, a beautifier or an indexer
      The recognizer is able to process the digital ink and output the recognition result, given the resources.

    When you plan to perform recognition with the MyScript SDK, it is absolutely vital to choose the resources properly. This choice highly impacts on the efficiency and the accuracy of the recognition.

    There are three main groups of resources:

    • Alphabet Knowledge resources (AK)
      These resources gather all the existing symbols for a target writing system. For instance, for math recognition, the existing symbols refer to math symbols. For English text recognition, the AK would provide the knowledge of the Latin alphabet plus some particular symbols (digits, punctuation marks and other characters).

    • Linguistic Knowledge resources (LK) or Grammar resources (GRM)
      These resources supply information on the ordering rules that confer meaning to a message. For instance, for text recognition, an LK resource provides the knowledge of the words, phrases or particular semantic units (such as dates, phone numbers and so on).

    • Subset Knowledge resources (SK)
      This type of resource defines a subset of the symbols existing in the AK. The Subset Knowledge reduces the set of symbols to recognize. This could be particularly helpful when you want to recognize a form field that accepts only numbers (for a postal code, an ID number, etc.).

    Attaching an AK is mandatory. LK is mostly used and SK is optional.

    Recognition process

    MyScript recognizer relies on three main processes taking place simultaneously: symbol classification, segmentation and linguistic-based analysis. Information is exchanged between the three processes to find the most probable recognition result.

    The symbol classification and the segmentation rely on the knowledge of the existing symbols related to an input form type (an input form can be a piece of text, or a shape, or an equation, or a music score etc.).

    The segmentation tries to cut or group strokes together. Each group hypothesis is meant to match as close as possible an existing symbol. For instance, it can group strokes to form a character, or a music note, or a shape and so on.

    The linguistic-based analysis relies on a set of norms that rule the way the existing symbols of an input form type can be combined to make up a meaningful content.

    Output

    The output of the recognition process is usually delivered in the form of a tree. The structure of this tree depends on the type of recognition that you run. In any case, it contains a list of symbols or groups of symbols that potentially match the input ink. These are referred to as candidates.

    The candidates are ranked in the list according to scoring metrics. Whenever possible, the recognizer will designate several candidates for a given portion of input ink and attribute scores to each. The ranking also depends on the constraints placed on the recognition process (resources, etc.). The recognizer will designate a selected candidate at each level in the tree result.

    Finally, input ranges link the recognition result elements to their counterparts in the input ink. An input range represents a portion of ink from the starting point in the starting stroke to the ending point in the ending stroke.

    When you send the input digital ink to the recognizer, the latter will process it and segment it into significant elementary ink portions, also called units. The recognizer does not perform the segmentation at random. In fact, it performs the cuts on purpose, as soon as they are meaningful in the sense of the attached resources.

    Depending on the input form (text, math expression, shape, etc.), the recognizer may consider units at different levels. A set of elementary units can belong to a macro unit, and so on. For instance, if you input a text, the smallest unit is the character. Then, sets of characters form a word, that is a another unit kind. And finally, sets of words form a sentence, that is another unit kind as well.

    At each level, every unit is expressed by its input range. The input range tells you the starting point of the ink portion and its ending point. To do so, each stroke of the input ink is indexed. Then each point of a single stroke is also indexed.
