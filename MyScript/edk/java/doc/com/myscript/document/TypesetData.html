Class TypesetData

  • All Implemented Interfaces:
    IDisposable, IEngineObject, java.lang.Cloneable

    public final class TypesetData
extends EngineObject
    The TypesetData class models a typeset data describing the outcome of a typeset operation.

    Please note that the implementation of the TypesetData class is not synchronized. If multiple threads concurrently try to change the state of a TypesetData object, explicit synchronization must be done externally.

      • apply

        public final void apply(boolean keepLayout,
         PageSelection oldData,
         PageSelection newData)
                 throws NotPermittedFromThisThreadException,
                        java.lang.NullPointerException,
                        java.lang.IllegalStateException
        Applies the typesetting modifications described in the given TypesetData. If items that already reference a typeset result have been modified, a new glyph or primitive will be created, and the old one will be added to oldData.
        Parameters:
        keepLayout - true if the layout information associated with the strokes to be typeset should be transferred to their typeset counterpart.
        oldData - a selection will be filled with the strokes that have been replaced by typeset data, or null.
        newData - a selection will be filled with the primitives or glyphs generated by the typeset operation, or null.
        Throws:
        NotPermittedFromThisThreadException - the apply(boolean, PageSelection, PageSelection) call was made from a callback of the associated content processor.
        java.lang.NullPointerException - when oldData is null .
        java.lang.NullPointerException - when newData is null .
        java.lang.IllegalStateException - when the ContentField associated with this TypesetData has been modified since this TypesetData was created (e.g. by applying a PageChangeSet or by processing that ContentField).
        java.lang.IllegalStateException - when this TypesetData object has been disposed of.
