    public abstract class Vocabulary
extends LinguisticKnowledge
    The Vocabulary class is the abstract base class that models vocabulary level linguistic knowledge given to the recognizer.

    Please note that the implementation of the Vocabulary class is not synchronized. If multiple threads concurrently try to change the state of a Vocabulary object, explicit synchronization must be done externally.

      • getReservedRecognitionPaths

        public final float getReservedRecognitionPaths()
                                        throws java.lang.IllegalStateException
        Returns the reserved recognition paths for this Vocabulary resource.

        The reserved recognition path is a value between 0 and 1, that sets aside a certain percentage of the paths that the recognizer explores for one particular resource. By doing this, the Vocabulary resource can supply a recognition candidate even if the scores from its paths are noticeably lower than the other scores. For example you could use frequency scale property in conjunction this property to give precedence to a resource but reserve some paths for another resource in order to provide a safety net in case of a nil match.

        Returns:
        the reserved recognition paths value (a float value between 0 and 1).
        Throws:
        java.lang.IllegalStateException - when this Vocabulary object has been disposed of.

        public final void setRecognitionPathLimit(float paths)
                                   throws java.lang.IllegalStateException,
                                          java.lang.IllegalArgumentException,
                                          ModificationAccessDeniedException
        Sets the recognition path limit for this Vocabulary resource.

        The recognition path limit is a value between 0 and 1, that sets aside a maximum percentage of the paths that the recognizer explores for one particular resource. By doing this, the Vocabulary resource will not populate the whole recognition path bandwidth. Using this property in conjunction with the setReservedRecognitionPaths(float) property gives complete control over the bandwidth for the Vocabulary resource.

        Parameters:
        paths - the recognition paths limit value (a float value between 0 and 1).
        Throws:
        java.lang.IllegalStateException - when this Vocabulary object has been disposed of.
        java.lang.IllegalArgumentException - when paths is Not-a-Number (NaN).
        java.lang.IllegalArgumentException - when paths is infinitely large in magnitude.
        java.lang.IllegalArgumentException - when paths is not a positive float value between 0 and 1.
        ModificationAccessDeniedException - when this Vocabulary object is in use by another object.

        public final CasingScheme getCasingScheme()
                                   throws java.lang.IllegalStateException
        Returns the current casing scheme that controls the way character case variations are being handled by the recognition process. By default, Lexicon and LexicalUnit resources use legacy casing rules (@see CasingScheme.LEGACY_CASE): 
        [Vocabulary entry] -> [What can be recognized]
 hello              -> hello, Hello, HELLO
 Hello              -> Hello, HELLO
 HELLO              -> HELLO
        Returns:
        a CasingScheme object.
