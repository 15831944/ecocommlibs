com.myscript.text

Class HandwritingContext

  • java.lang.Object
    • com.myscript.engine.EngineObject
      • com.myscript.text.HandwritingContext
  • All Implemented Interfaces:
    IAttachTarget, IDisposable, IEngineObject, IInput, IInputConsumer, java.lang.Cloneable
    Direct Known Subclasses:
    DigitalInkTagger, Indexer, Recognizer

    public abstract class HandwritingContext
extends EngineObject
implements IAttachTarget, IInputConsumer
    The HandwritingContext class models a MyScript technology entity that is able to deal with handwriting.

    Please note that the implementation of the HandwritingContext class is not synchronized. If multiple threads concurrently try to change the state of a HandwritingContext object, explicit synchronization must be done externally.

        public static final void setUserLimit(Engine engine,
                int limit)
                               throws java.lang.NullPointerException,
                                      java.lang.IllegalStateException,
                                      java.lang.IllegalArgumentException
        Sets the maximum number of HandwritingContext objects that can exist at the same time in the user side of a MyScript Engine enabled application.

        The special value zero means no limit. Other values are adjusted to the next multiple of 16. A value under the current count is permitted and will constrain the future HandwritingContext object creations.

        Parameters:
        engine - an Engine object.
        limit - the value of the new creation limit.
        Throws:
        java.lang.NullPointerException - when engine is null.
        java.lang.IllegalStateException - when engine has been disposed of.
        java.lang.IllegalArgumentException - when limit < 0.
