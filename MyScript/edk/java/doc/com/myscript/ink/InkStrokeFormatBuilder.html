Class InkStrokeFormatBuilder

  • java.lang.Object
    • com.myscript.engine.EngineObject
      • com.myscript.ink.InkStrokeFormatBuilder
  • All Implemented Interfaces:
    IDisposable, IEngineObject, IInkStrokeFormat, java.lang.Cloneable

    public final class InkStrokeFormatBuilder
extends EngineObject
implements IInkStrokeFormat
    A InkStrokeFormatBuilder is used to build InkStrokeFormat objects by specifying their properties and then invoking createStrokeFormat(). A InkStrokeFormatBuilder can be recycled to create other stroke formats by invoking clear() between each. Alternatively you can use the static factory parseInkStrokeFormat(Engine, String) to create a stroke format by parsing its string representation, without the need to create an instance of the InkStrokeFormatBuilder class.

    For example:

     InkStrokeFormatBuilder strokeFormatBuilder = InkStrokeFormatBuilder.create(engine);
 strokeFormatBuilder.addChannel("X");
 strokeFormatBuilder.addChannel("Y");
 strokeFormat1 = strokeFormatBuilder.createStrokeFormat();
 strokeFormatBuilder.dispose();

    and

     strokeFormat2 = InkStrokeFormatBuilder.parseInkStrokeFormat("{\"channels\":[{\"name\":\"X\"},{\"name\":\"Y\"}]}");

    will both create a stroke format that contains only X and Y channels with all remains parameters left to their default values.

    Note that the native side ensures that two stroke format with the same intrinsic parameters will be returned as the same instance. Therefore, strokeFormat1.isSame(strokeFormat2) will be true from the above examples and we implemented InkStrokeFormat.equals(Object) by using it.

      • addChannel

        public void addChannel(java.lang.String name)
        Adds a new channel to the stroke format to build.

        Here below are the predefined names and their recognized meaning:

        Name Description
        X Horizontal position, increasing to the right.
        Y Vertical position, increasing downward.
        W Stroke width.
        T Sample timestamp.
        F Tip force.
        OTx Tilt along the X axis.
        OTy Tilt along the Y axis.
        OA Azimuth angle (yaw).
        OE Elevation angle (pitch).
        OR Rotation (counter-clockwise rotation around the pen axis).

        You can also store application specific values by using a name not found in the table above.

        Please also note that a stroke format must at least contain X and Y channels.

        Parameters:
        name - the name of the channel to add.
