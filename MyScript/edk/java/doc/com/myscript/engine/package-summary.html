Package com.myscript.engine

The com.myscript.engine package provides classes and interfaces that model the generic access to the MyScript Engine technology.

See: Description

    IAttachTarget
    The IAttachTarget interface models a target EngineObject to which other EngineObject can be attached.
    ICompilable
    The ICompilable interface models an EngineObject that is compilable, in other words on which you can invoke the compile and sometimes decompile methods.
    IDisposable
    The IDisposable interface defines the dispose method that allows you to perform tasks associated with freeing, releasing or resetting unmanaged resources allocated from the native side of the MyScript Engine.
    IEngineObject
    The IEngineObject interface defines the getEngine and the getNativeReference methods that allow interaction with the native side of the MyScript Engine.
    IFloatStroke
    The IFloatStroke interface is meant to be implemented by the client code in order to feed the handwriting recognition engine with strokes whose coordinates are represented in the float number format.
    IFloatStructuredInput
    The IFloatStructuredInput interface models a structured input to the MyScript technologies where the parameters are expressed in the float number format.
    IInput
    The IInput interface models an input to the MyScript technologies.
    IInputConsumer
    The IInputConsumer interface models an EngineObject object that can receive an input.
    IProgress
    The IProgress interface models a progress callback used when the the MyScript Engine needs to notify the client side about the status of a time consuming operation being performed.
    IRunnable
    The IRunnable interface models an EngineObject that is able to run a treatment, in other words on which you can invoke the run method.
    IStroke
    The IStroke interface models the common services of all the different stroke types.
    IStructuredInput
    The IStructuredInput interface models a structured input to the MyScript technologies.
    IVersionable
    Represents a versionable object.

Package com.myscript.engine Description

The com.myscript.engine package provides classes and interfaces that model the generic access to the MyScript Engine technology.

When using the MyScript technologies, you need the com.myscript.engine package uses JNI (Java Native Interface) to load its native side library counterpart called "MyScriptEngine" (exact naming depends on the target platform, eg MyScriptEngine.dll on Windows platforms, libMyScriptEngine.so on Linux platforms).

The important thing to know about using JNI under Tomcat is that one cannot place the native libraries OR their JNI part under the WEB-INF/lib or WEB-INF/classes directories of a web application and expect to be able to reload the webapp without restarting the server: this is because with JNI, the same JNI native library cannot be loaded into more than one class loader. The solution to this problem is to place the MyScript Engine native libraries into the TOMCAT/shared/lib directory.
