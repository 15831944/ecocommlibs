com.myscript.engine

Class Engine

  • java.lang.Object
    • com.myscript.engine.Engine
  • All Implemented Interfaces:
    IDisposable

    public final class Engine
extends java.lang.Object
implements IDisposable
    The Engine class models the MyScript Engine.

    In order to use the various MyScript technologies, you first have to obtain an instance of the Engine class by invoking the create static factory method.

    RULE 1: you install it, you dispose of it!

    This is really a very important rule, because of the native implementation of the MyScript technologies: you must explicitly dispose of the Engine objects you obtain.

    Once you have obtained an object of the Engine class, you are able to create or load EngineObject objects, objects that expose aspects of the different MyScript technologies.

    RULE 2: disposing of a parent engine disposes of the children !

    All the EngineObject objects you create or load are linked with an Engine object: in other words, any EngineObject object is the child of an Engine object. As a consequence, when disposing of an Engine object, you also dispose of all its children and any Java references of these children you are still holding become unusable: trying to use them will result in a IllegalStateException exception being thrown.

        public static final void setLimit(int limit)
                           throws java.lang.IllegalArgumentException
        Sets the maximum number of Engine objects that can exist at the same time in a MyScript enabled application.

        The special value zero means no limit. Other values are adjusted to the next multiple of 16. A value under the current count is permitted and will constrain the future Engine object creations.

        Parameters:
        limit - the value of the new creation limit.
        Throws:
        java.lang.IllegalArgumentException - when limit < 0.
