How This API Document Is Organized

This API (Application Programming Interface) document has pages corresponding to the items in the navigation bar, described as follows.

Overview

The Overview page is the front page of this API document and provides a list of all include files with a summary for each. This page can also contains an overall description of the whole API.

Include Files

Each include file has a page that contains a list of the objects it defines with a summary for each. This page can contain five categories:

  • types (simple types, classes and interfaces)
  • enumerations
  • structures
  • defines
  • functions

Simple Types, Classes and Interfaces

Each type has its own separate page. Each of these pages has three sections consisting of a type/class/interface description, summary table, and detailed member descriptions:

    only for Classes
  • Class inheritance diagram
  • Type Properties*
  • Properties*
  • All Implemented Interfaces
  • Direct Known Subtypes

    only for Interfaces
  • Class inheritance diagram
  • Direct Known Subtypes

    for every Type
  • Include file to use
  • Type description
  • Members Summary
  • Members Detail
Each summary entry contains the first sentence from the detailed description for that item. The summary entries are alphabetical, while the detailed descriptions are in the order they appear in the source code. This preserves the logical groupings established by the programmer.

Enumeration

Each enumeration has its own separate page with the following sections:

  • Enumeration declaration
  • Enumeration description

Type Properties

The Type Properties item contains link(s) to enumeration(s) that list the properties applicable to the type. A type property can be seen as a static member variable in a C++ class. Accessing a type property with id $PROPERTY with value type $value_t applied to a type with id $TYPE (e.g.: the id of the type voMyType is VO_MyType) is done as follows: 

$value_t value;
voGetTypeProperty(engine, $TYPE, $PROPERTY, &value, sizeof(value));
voSetTypeProperty(engine, $TYPE, $PROPERTY, &value, sizeof(value));

Properties

The Properties item contains link(s) to enumeration(s) that list the properties applicable to the instances of the type. A type property can be seen as a member variable in a C++ class. Accessing a property with id $PROPERTY with value type $value_t applied to an object is done as follows: 

$value_t value;
voGetProperty(engine, object, $PROPERTY, &value, sizeof(value));
voSetProperty(engine, object, $PROPERTY, &value, sizeof(value));
